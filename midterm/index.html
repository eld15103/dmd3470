WHAT IS DARK MODE?

Recently, dark mode has become increasingly popular on mobile devices and even certain destktop applications. Dark mode utilizes a darker background with lighter text. Dark Mode allows for websites to be more accessible for people with light sensitivity or vision problems. With how heavy technology plays a role in todayâ€™s society, it is important to make the user experience as accessible as possible. If a user has their browser settings set to a dark mode preference, websites that have implemented a dark mode, such as this one, will automatically be viewed in that preference. Websites can also provide a toggle that allows users to go between the two themes without changing their settings.

DESIGNING A DARK THEME

A proper dark theme is more than just the inverse of the light theme colors.
There are four main properties to keep in mind:

Contrast

Dark themes should have a contrast high contrast level.

Depth

Express elevation and space by creating depth.

Desaturation

Primary colors should be desaturated.

Limited Color

Limit the amount of color accents.

Dark themes are built using layers. One layer is the background, this layer is usually #121212. It is important to note that if designing for Apple, Apple standards are a background of #000000. A good way to remember this is that "Only a Sith deals in absolutes." Only Apple tends to use a pure black for their background color. The other layer is a semi-transparent overlay. When choosing an overlay, there is the option of white or a shade of color. This shade will be referred to as the primary. In dark mode, the primary color will shift to a desaturated color shade. Choosing a color shade can help to keep the brand image while in dark mode.

CODING A DARK THEME

Firstly, make sure your website is fully created in HTML before adding the dark theme through CSS. This will make it much easier to keep track of variables. There are two ways to go about implementing a dark theme.

The first way to implement a dark theme is to automatically do it for the user. To determine which theme the website should display to the user upon first loading, the user's settings will be read. This can be done by creating a media query. Prefers-color-scheme checks if your user preferes light or dark off of their browser settings. Based off of those settings, you can set certain settings for both themes. Keep in mind, only certain browsers allow you to read the user's settings. 

      @media (prefers-color-scheme: light){
          h1{
            color: black;
          }
        }
        @media (prefers-color-scheme: dark){
          h1{
            color: white;
          }
        }

If you would rather make a toggle between the light and dark mode, the easiest way is to use a tiny bit of JavaScript. JavaScript allows for a function to be made that can determine whether or not the dark mode has been activated. When the toggle is switched, dark mode will be activated. When dark mode is activated, all the CSS variables for dark mode will be activated. If our toggle class name were to be called dark-mode, the CSS code would look like: 

      h1{
          color: black;
        }
        .dark-mode h1{
          color: white;
        }

Now to add in the primary color and create depth, we use rgba instead of hex values. RGBA allows us to control the alpha transparency through the "a" value. If white is the primary color, the hex value #FFFFFF will be converted to RGB(255,255,255). For the alpha value, the higher the percentage, the more it is seen. In dark mode, the more emphasized areas should have a 6% alpha value, and the less emphasized can have a 3%. Notice how on this website, the header and footer have a 6% alpha value, whereas the body has 3%. The font color should also be converted into rgba in dark mode. For emphasized text, the alpha value should be 87%. If there is any text that does not require high empahsis, consider dropping it to a medium emphasis of a 60% alpha value. 

      .dark-mode body{
          background-color: #121212;
        }
  
        .dark-mode h1{
          color: (255,255,255,87%);
          background-color: (255,255,255,6%)
        }
